## Directory Purposes

**eval-lib/src/types/:**
- Core type system: Document, Corpus, CharacterSpan, PositionAwareChunk
- Branded types for compile-time type safety (DocumentId, QueryId, etc.)
- Zod schemas for validation (DocumentSchema, DatasetExampleSchema)

**eval-lib/src/chunkers/:**
- Implementations of chunking strategies
- All implement `Chunker` or `AsyncPositionAwareChunker` interface
- Must return `PositionAwareChunk[]` with exact character offsets

**eval-lib/src/synthetic-datagen/:**
- Question generation orchestration (generate() function)
- Three strategy implementations (Simple, DimensionDriven, RealWorldGrounded)
- GroundTruthAssigner for mapping questions to source document character spans

**eval-lib/src/evaluation/:**
- Metric computation on character-level spans
- Pure functions (recall, precision, IoU, F1)
- Span utilities (mergeOverlappingSpans, calculateOverlap, totalSpanLength)

**backend/convex/crud/:**
- CRUD operations for all domain entities
- All queries/mutations auth-scoped via getAuthContext()
- Org-based filtering applied to all reads/writes

**backend/convex/generation/:**
- Question generation domain logic
- Orchestration: Job creation, status, cancellation
- Actions: Strategy instantiation, question insertion callbacks

**backend/convex/retrieval/:**
- Document chunking and indexing domain
- Orchestration: Job creation, WorkPool setup
- Actions: Two-phase indexing, chunk embedding, vector search

**backend/convex/experiments/:**
- Experiment evaluation domain
- Orchestration: Experiment CRUD, status, results
- Actions: LangSmith runner, metric computation, result persistence

**backend/convex/lib/:**
- Shared helpers (auth context, validators, WorkPool, vector search)
- Auth extraction and org-scoped query helpers

**frontend/src/app/:**
- One page per major feature (kb, generate, experiments, retrievers)
- Each page is a Suspense boundary with reactive Convex queries
- Uses useQuery/useMutation for real-time updates

**frontend/src/components/:**
- Reusable React components (inputs, modals, viewers, selectors)
- Stateless presentation components + hooks for Convex integration
- Tailwind CSS dark theme (accent: #6ee7b7)

## Key File Locations

- `packages/frontend/src/app/layout.tsx`: Root layout wrapping all pages
- `packages/frontend/src/app/page.tsx`: Home page routing to ModeSelector

**Core Logic:**

**Testing:**

- `packages/eval-lib/tests/langsmith/`: LangSmith integration tests
- `packages/backend/tests/helpers.ts`: Shared Convex test fixtures
- `packages/backend/tests/`: Integration tests for Convex functions

## Naming Conventions

**Files:**
- TypeScript/TSX files: camelCase (e.g., `utils.ts`, `DocumentViewer.tsx`)
- Components: PascalCase (e.g., `AuthGate.tsx`, `GenerateConfig.tsx`)
- Tests: `name.test.ts` or `name.spec.ts`
- Config files: lowercase with extensions (e.g., `tsup.config.ts`, `convex.config.ts`)

**Directories:**
- Feature/domain directories: kebab-case (e.g., `synthetic-datagen`, `vector-stores`)
- Component directories: camelCase (e.g., `tabs`, `wizard`, `steps`)
- Organizational layers: lowercase descriptive (e.g., `crud`, `generation`, `retrieval`)

**Functions/Types:**
- Types: PascalCase (Document, Corpus, CharacterSpan, Retriever)
- Functions: camelCase (createDocument, generate, computeMetrics)
- Constants: UPPERCASE_SNAKE_CASE (BASELINE_VECTOR_RAG_CONFIG, DEFAULT_INDEX_CONFIG)
- Interfaces: PascalCase with prefix "I" or suffix "Interface" (Chunker, Embedder)

**Variables:**
- React state: camelCase (selectedKbId, questionsData, isLoading)
- DOM/CSS classes: kebab-case (via Tailwind: bg-bg, text-text, dark)

## Where to Add New Code

**New Feature (full flow):**
- Domain logic: Backend mutation/query in `packages/backend/convex/[domain]/orchestration.ts`
- Long-running work: Action in `packages/backend/convex/[domain]/actions.ts` enqueued via WorkPool
- Database persistence: Add table to `packages/backend/convex/schema.ts`
- Frontend page: `packages/frontend/src/app/[feature]/page.tsx`
- Frontend components: `packages/frontend/src/components/[FeatureName].tsx`
- Library type: `packages/eval-lib/src/types/` + re-export from `index.ts`

**New Chunker/Strategy/Embedder:**
- Implementation: Create file in `packages/eval-lib/src/[chunkers|synthetic-datagen/strategies|embedders]/`
- Interface compliance: Implement required interface (Chunker, QuestionStrategy, Embedder)
- Tests: Mirror directory structure in `packages/eval-lib/tests/unit/`
- Export: Re-export from `packages/eval-lib/src/[category]/index.ts` → main index.ts

**New Utility Function:**
- Shared across packages: `packages/eval-lib/src/utils/`
- Backend-only: `packages/backend/convex/lib/`
- Frontend-only: `packages/frontend/src/lib/`

**Database CRUD:**
- Query/mutation: `packages/backend/convex/crud/[entity].ts` (org-scoped via getAuthContext)
- Internal operations: `packages/backend/convex/[domain]/` (called by actions or orchestration)

**Test:**
- Unit test (eval-lib): `packages/eval-lib/tests/unit/[category]/name.test.ts`
- Integration test (backend): `packages/backend/tests/name.test.ts` (uses setupTest helper)

## Special Directories

**packages/backend/convex/_generated/:**
- Purpose: Auto-generated Convex types and API
- Generated: Yes (by `npx convex dev`)
- Committed: Yes (checked into git for type safety)

**packages/eval-lib/dist/:**
- Purpose: Compiled JavaScript + types (tsup output)
- Generated: Yes (by `pnpm build`)
- Committed: No (gitignored, built on install)

**packages/frontend/.next/:**
- Purpose: Next.js build cache and compiled app
- Generated: Yes (by `next build` or `next dev`)
- Committed: No (gitignored)

**data/:**
- Purpose: Sample markdown files, test corpora
- Generated: No (static data)
- Committed: Yes (for reproducible evaluation examples)
